# Health Tracker — Supabase 同期化 設計

作成日: 2026-06-05

## 目的

現在 localStorage に保存している Health Tracker（HTML 1枚の SPA）を、
**スマホ・PC 間でデータが同期される**形にする。スマホからアイコンタップで簡単に入力できるようにする。

## 前提・制約

- **利用者**: 本人のみ（個人利用）
- **コスト**: 完全無料（GitHub Pages + Supabase 無料枠）
- **接続**: 常にオンライン前提（オフライン入力は対象外）
- **既存データ移行**: 不要（作り直しで良い）
- アプリは **今の `index.html` 1枚構成のまま**。保存先と起動時処理のみ変更。

## アプローチ

**Supabase（Postgres + Auth）＋ マジックリンク認証 ＋ RLS** を採用。

- データは Supabase の Postgres に保存し、端末間で同期
- ログインはメールに届くリンクをタップ（パスワード不要）。各端末で初回1回のみ
- Row Level Security により「自分の行だけ」アクセス可能にし、健康データを保護
- ホスティングは GitHub Pages（静的1ファイル）

却下案:
- PINのみ（認証なし）→ 公開キー＋URLでデータが他人に読まれ得るため健康データに不適
- Firebase → NoSQL が現データ構造（日付・テーブル）と相性△、設定もやや複雑

## 全体構成

```
[スマホ/PCのブラウザ]
   index.html (GitHub Pagesで公開)
   └─ supabase-js (CDN読み込み)
        │  ① ログイン（マジックリンク）
        │  ② データ読み書き
        ▼
[Supabase 無料枠]
   ├─ Auth（メール認証・セッション管理）
   └─ Postgres（データ保存）＋ RLSで本人だけアクセス可
```

## データ構造（Supabase テーブル）

| テーブル | 列 | 制約 |
|---------|-----|------|
| **body_records** | id, user_id, date, weight, body_fat | UNIQUE(user_id, date) |
| **exercise_records** | id, user_id, date, type | UNIQUE(user_id, date, type) |
| **tasks** | id, user_id, name, status, date | id はアプリ生成の uuid |

- `type` は text。値: `筋トレ / RUN / WALKING / BIKE / HIIT`（text なので後から種目追加可能）
- 運動は「その日にやった種目の行が存在する＝やった」方式。1日に複数種目OK。
- 全テーブルに `user_id`（auth.uid()）。RLS で `user_id = auth.uid()` の行のみ select/insert/update/delete 可。

## ログインの流れ（マジックリンク）

```
初回（端末ごとに1回）
  アプリを開く → ログイン画面（メール入力）
    → メールのリンクをタップ → アプリに戻る＝ログイン完了 → データ表示

2回目以降
  アプリを開く → セッションが残存 → 即データ表示（ログイン不要）
```

- セッションは supabase-js がブラウザに保持。毎回ログインは不要。
- セッション切れ時はログイン画面に戻す。

## 既存コードの改修ポイント

| 箇所 | 変更内容 |
|------|---------|
| **DataManager** | `load/save`（localStorage 同期）を Supabase の読み書きに置換。各メソッドを `async` 化 |
| **呼び出し側（描画）** | 同期前提だった箇所を `await` 対応に修正 |
| **起動時** | ①セッション確認 → ②未ログインならログイン画面 → ③ログイン済みなら全データ取得して描画 |
| **運動UI** | 3ボタン（筋トレ/有酸素/HIIT）→ 5種目トグル（筋トレ/RUN/WALKING/BIKE/HIIT）。押すとその日の該当行を ON/OFF |
| **設定追加** | Supabase の URL・公開（anon）キーを HTML 内に記載 |

見た目・操作はほぼ現状維持。裏側の保存先と、起動時のログイン処理が主な差分。

## エラー処理（常時オンライン前提）

| 状況 | 挙動 |
|------|------|
| ネット未接続 / Supabase 応答なし | 「接続できません。通信環境を確認してください」を表示。入力中の値は消さない |
| 保存失敗 | エラー通知＋再試行可能に。入力値は保持 |
| セッション切れ | ログイン画面に自動で戻す |

## テスト（手動・実機確認）

1. PC でログイン → 体重・運動・タスクを入力
2. スマホで開く → 同じデータが見える（同期検証）
3. スマホで入力 → PC で再読込 → 反映される
4. 運動5種目のトグル ON/OFF が保存される
5. ログアウト後、別 URL アクセスでデータが見えない（RLS 確認）

## セットアップ手順（初回のみ）

1. Supabase で無料プロジェクト作成
2. SQL でテーブル3つ＋RLSポリシー作成
3. Auth のメール認証を有効化＋リダイレクト URL 設定
4. `index.html` に Supabase の URL・公開キーを記入
5. GitHub に push → GitHub Pages 有効化 → 公開 URL 発行
6. スマホで URL を開き「ホーム画面に追加」
